# 📖 Руководство пользователя AutoSend Letters HeadHunter

Полное руководство по использованию скрипта автоматической отправки откликов на вакансии HeadHunter с поддержкой AI-генерации сопроводительных писем.

## 🎯 Возможности скрипта

### ✅ Что уже реализовано (версия 2.2.0+)

- **Автоматическая отправка откликов** на вакансии
- **Выбор резюме** для отклика через удобный интерфейс
- **Настройка времени задержки** между действиями
- **Множественные шаблоны** сопроводительных писем
- **🆕 AI-генерация персонализированных писем** (ChatGPT, Yandex GPT, GigaChat, Gemini)
- **🆕 Удобный интерфейс настроек** без редактирования кода
- **Обработка различных типов вакансий** (обычные, из других городов, непрямые работодатели)
- **Автоматический переход** к следующей вакансии после отклика

### 🚧 В разработке

- **Автоматическое обновление резюме** - поднятие в поисковой выдаче
- **Массовая отправка откликов** на множество вакансий одновременно
- **Статистика откликов** и аналитика эффективности

## 🚀 Быстрый старт

### Для пользователей (рекомендуется)

1. **Установите расширение Scripty** для Chrome:
   - Перейдите по ссылке: [Scripty Extension](https://chrome.google.com/webstore/detail/scripty-javascript-inject/milkbiaeapddfnpenedfgbfdacpbcbam)
   - Нажмите "Добавить в Chrome"

2. **Установите скрипт автоматически**:
   - Перейдите по ссылке: [Установить скрипт](https://scripty.abhisheksatre.com/#/share/script_mbb7h4qpayicsv)
   - Нажмите "Install Script"

3. **Откройте HeadHunter** и настройте скрипт:
   - Перейдите на [hh.ru](https://hh.ru)
   - В верхней навигации появится кнопка "⚙️ Настройки AutoSend"
   - Нажмите на неё для открытия настроек

### Для разработчиков

1. **Клонируйте репозиторий**:
   ```bash
   git clone https://github.com/1MaxSpb/autosend-letters-hh.git
   cd autosend-letters-hh
   ```

2. **Установите зависимости**:
   ```bash
   npm install
   ```

3. **Соберите проект**:
   ```bash
   npm run build
   ```

4. **Используйте файл** `dist/script.js` в браузере

## ⚙️ Настройка скрипта

### Базовые настройки

1. **Откройте настройки**:
   - Нажмите "⚙️ Настройки AutoSend" в навигации HeadHunter

2. **ID резюме**:
   - Найдите ID вашего резюме (см. раздел "Как найти ID резюме")
   - Введите в поле "ID резюме"

3. **Задержка между действиями**:
   - Рекомендуется: 1500-3000 мс
   - Меньше 1000 мс может привести к блокировке
   - Больше 5000 мс замедлит работу

4. **Шаблон письма**:
   - Выберите из готовых шаблонов
   - Или создайте свой (см. раздел "Создание шаблонов")

### AI настройки

1. **Включите AI генерацию**:
   - Поставьте галочку "Включить AI генерацию"
   - Откроются дополнительные настройки

2. **Выберите провайдера**:
   - ChatGPT - лучшее качество, платно
   - Yandex GPT - хорошо для русского языка
   - GigaChat - бесплатные лимиты
   - Gemini - быстрый и качественный

3. **Введите API ключ**:
   - Получите ключ у выбранного провайдера
   - Подробная инструкция в [AI_SETUP.md](AI_SETUP.md)

4. **Сохраните настройки**:
   - Нажмите "💾 Сохранить настройки"

## 📝 Как найти ID резюме

### Способ 1: Через URL резюме
1. Откройте ваше резюме на HeadHunter
2. Скопируйте URL (например: `https://hh.ru/resume/12345678`)
3. ID резюме - это число в конце URL (`12345678`)

### Способ 2: Через инспектор элементов
1. Откройте любую вакансию на HeadHunter
2. Нажмите "Откликнуться"
3. Откройте инспектор элементов (F12)
4. Найдите выпадающий список с резюме
5. Посмотрите атрибут `data-qa` - там будет ваш ID

## 📄 Создание и редактирование шаблонов

### Готовые шаблоны

Скрипт поставляется с несколькими готовыми шаблонами:
- `coverLetter_1` - профессиональный стиль
- `coverLetter_2` - краткий формат
- `coverLetter_3` - творческий подход
- `coverLetter_4` - технический фокус
- `coverLetter_5` - универсальный

### Создание собственного шаблона

1. **Откройте файл** `src/data/data.js`
2. **Добавьте новый шаблон**:
   ```javascript
   export const TEMPLATES = {
     // существующие шаблоны...
     
     coverLetter_custom: `Ваш персональный шаблон здесь.
     
     Используйте {#vacancyName} для автоматической подстановки названия вакансии.
     
     С уважением, Ваше имя`
   };
   ```
3. **Пересоберите проект** командой `npm run build`

### Переменные в шаблонах

- `{#vacancyName}` - автоматически заменяется на название вакансии

## 🎯 Эффективное использование

### Лучшие практики

1. **Настройте задержки правильно**:
   - 1500-2000 мс для быстрой работы
   - 3000-5000 мс для безопасности

2. **Используйте AI для персонализации**:
   - Включите AI генерацию для лучших результатов
   - Разные провайдеры дают разный стиль письма

3. **Мониторьте работу**:
   - Следите за консолью браузера (F12)
   - Проверяйте отправленные отклики

4. **Делайте перерывы**:
   - Не отправляйте более 50 откликов подряд
   - Используйте скрипт в разное время дня

### Типичные ошибки

❌ **Неправильный ID резюме** - отклики не отправляются  
✅ Проверьте ID через URL резюме

❌ **Слишком маленькая задержка** - блокировка аккаунта  
✅ Используйте задержку не менее 1500 мс

❌ **Неверный API ключ** - AI не работает  
✅ Проверьте ключ и баланс у провайдера

❌ **Массовые отклики без разбора** - бан от HeadHunter  
✅ Отбирайте вакансии и делайте перерывы

## 🛠️ Устранение проблем

### Скрипт не работает

1. **Проверьте консоль**:
   - Нажмите F12 → Console
   - Найдите ошибки (красный текст)

2. **Обновите страницу**:
   - Ctrl+F5 для полного обновления
   - Проверьте, что Scripty активно

3. **Переустановите скрипт**:
   - Удалите из Scripty
   - Установите заново

### Отклики не отправляются

1. **Проверьте ID резюме**:
   - Убедитесь, что ID правильный
   - Резюме должно быть опубликовано

2. **Проверьте селекторы**:
   - HeadHunter может изменить интерфейс
   - Обновите скрипт до последней версии

### AI не генерирует письма

1. **Проверьте настройки**:
   - AI должен быть включен
   - API ключ должен быть валидным

2. **Проверьте баланс**:
   - У большинства провайдеров платное API
   - Пополните баланс при необходимости

3. **Попробуйте другого провайдера**:
   - Разные провайдеры могут работать по-разному
   - Yandex и GigaChat имеют бесплатные лимиты
